테스트 설명용 애노테이션 만들기
    package restapi/common

    new interface TestDescription
        @Target(ElementType.METHOD)
        @Retention(RetentionPolicy.SOURCE)  // 컴파일 이후는 사라짐
        public @interface TestDescription {
            String value(); // 기본 값 없으므로 항상 값이 있어야 한다.
        }

@TestDescription("정상적으로 이벤트를 생성하는 테스트")
@TestDescription("입력받을 수 없는 값을 사용해서 이벤트를 생성하는 경우에 에러가 발생하는 테스트")
@TestDescription("입력값이 비어있는 경우 이벤트를 생성하는 경우에 에러가 발생하는 테스트")
@TestDescription("입력값이 잘못된 경우 이벤트를 생성하는 경우에 에러가 발생하는 테스트")

JUnit5 에서는 이미 이런 애노테이션이 만들어져 있으니 사용하면 된다.

애노테이션을 주석 대신에 사용한 것!
