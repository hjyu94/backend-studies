31. 스프링 데이터 JPA: JPA Repository

@EnableJpaRepositories
    ● 스프링 부트 사용할 때는 사용하지 않아도 자동 설정 됨.
    ● 스프링 부트 사용하지 않을 때는 @Configuration과 같이 사용.
    (그래야 JpaRepository 인터페이스를 상속 받아 사용할 수 있다)
    ● 이 애노테이션에 설정할 수 있는 값들
    - entityManager
    - transactionManager
    - basePackages
        (패키지 스캔을 시작할 기본 패키지, 이 패키지부터 시작해서 Repository 빈을 찾는다.
        기본적으로는 @SpringBootApplication 위치부터 찾는다)

@Repository 애노테이션을 붙여야 하나 말아야 하나...
    ● 안붙여도 됩니다.
    ● 이미 붙어 있어요. 또 붙인다고 별일이 생기는건 아니지만 중복일 뿐입니다.
    SimpleJpaRepository 가 JpaRepository 의 실제 구현체가 되는데 Simple... 에 이미 @Repository가 붙어있다.

스프링 @Repository
    ● SQLExcpetion 또는 JPA 관련 예외를 스프링의 DataAccessException으로 변환 해준다. (스프링 프레임워크의 기능)
    (예외 클래스를 좀 더 계층화. 구체화해서 예외의 원인을 찾기 쉽게 하려는 목적)
